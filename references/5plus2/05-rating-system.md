# 05 · Rating System (the productized conclusion layer)

> This describes the **rating framework** the 5+2 method feeds into. It does **not**
> reproduce any specific published ratings — only the structure and the calibration
> lessons, so the skill can produce a consistent card for names you do cover.

## Two layers
- **Free 5+2 write-up = understanding layer.** Ends at "fairly valued + valuation range +
  risks to monitor." Its author explicitly says this is "the start of understanding, not a
  buy/sell signal."
- **Paid/tracking product = conclusion layer.** Emits a structured card: **rating + tier +
  target price + bear–bull range**.

🔴 **Coverage gate.** Only emit a rating card for a name that is actually in your tracking
universe. For a name you don't cover, stop at the understanding layer (valuation range +
risks). If the user explicitly asks for a prediction on an uncovered name, label it
"⚠️ prediction, not yet covered."

## Card fields
1. **Rating**: STRONG BUY / BUY / HOLD (no SELL in the covered set — coverage skews to names
   the author likes/tracks).
2. **Tier**: 1 / 2 / 3 / 4.
3. **Target price** (single number).
4. **Target range** (bear–bull, = the DCF bear/bull scenarios).
5. Current price + day change.

## How to read the two dials (decoded, not stated by him)
### Tier = risk/certainty (1 = safest/deepest moat … 4 = most speculative)
Driven by ⑦ risk + moat depth + balance-sheet safety:
- **Tier 1** (highest certainty): deep-moat / chokepoint / defensive names.
- **Tier 2**: quality large-caps (high certainty, some cyclicality/competition/complexity).
- **Tier 3**: higher risk — rich-multiple / capital-black-hole / strong-cyclical / story /
  turnaround.
- **Tier 4** (most speculative): loss-making / highest uncertainty.
→ The tier is the **risk axis** of "valuation ⊥ risk", productized.

### Rating = upside-to-target × conviction (not purely mechanical)
- **STRONG BUY** ≈ high upside (≥~28%) + high conviction.
- **BUY** = moderate upside, OR high upside suppressed by risk (still worth holding).
- **HOLD** = limited upside (target nearly reached) and/or thesis unproven.
→ A high-quality name with low upside can still be a BUY; low upside + unproven thesis → HOLD.

## Calibration lessons (apply these)
- **Target ∈ [DCF base, bull]** — NOT "target ≈ DCF base". On a high-conviction tier-1 name the
  target can sit *above* the DCF base (e.g. target above base, with the bear/bull range equal to
  the DCF bear/bull). On a tier-2 name the target may hug the base. Don't conflate target with base.
- **Anchor the DCF base BELOW the analyst mean** — his DCF tends to run more conservative than the
  sell-side mean.
- **The #1 named risk does not necessarily cut the rating** — it shows up in the *tier* (risk axis),
  not the *rating*. A name can have a clear top risk and still be STRONG BUY if the bull case and
  valuation support it.
- **Bear/bull range = DCF bear/bull scenarios** — keep them consistent with the ⑤ DCF.

## Calibration discipline (from running the method "blind then compared")
- The qualitative skeleton and the SaaS quantitative playbook generalize well out-of-sample.
- The two recurring biases to watch: (a) DCF base set too high — fix by anchoring below the analyst
  mean; (b) calling a premium-multiple SaaS "expensive" too quickly — fix by a quality-adjusted peer
  comp (the Meritech 20%-growth + 20%-FCF rule) before concluding.
- Always check: covered name → card; uncovered → understanding layer only.
